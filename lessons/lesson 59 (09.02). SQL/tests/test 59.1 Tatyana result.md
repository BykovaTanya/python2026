# Вопрос 1. Тема: Основы SQL
  
**1. Какая команда используется для извлечения всех записей из таблицы в SQL?**
  
Варианты ответов:
1) SELECT FROM table_name;
2) SELECT all FROM table_name;
3) ✅ SELECT * FROM table_name;
4) SELECT 1 FROM table_name;
  
### ИИ анализ
Правильный ответ — `SELECT * FROM table_name;`. Символ `*` является специальным оператором, который означает "все столбцы". Другие варианты синтаксически неверны или выбирают конкретные значения, а не все записи. Понимание этого оператора — ключевой элемент для написания базовых SQL-запросов.  

# Вопрос 2. Тема: Основы SQL
  
**2. Как ограничить выборку по конкретному условию?**
  
Варианты ответов:
1) ✅ SELECT * FROM table_name WHERE condition;
2) SELECT * FROM table_name ORDER BY condition;
3) SELECT WHERE condition FROM table_name;
4) SELECT * WHERE condition FROM table_name;
  
### ИИ анализ
Правильный ответ использует ключевое слово `WHERE` для фильтрации строк по условию. Важно помнить порядок предложений в SQL: `SELECT` указывает столбцы, `FROM` — таблицу, а `WHERE` — условие фильтрации, которое должно следовать после `FROM`. Другие варианты нарушают этот порядок или используют неверные ключевые слова.  

# Вопрос 3. Тема: Основы SQL
  
**3. Как отсортировать результаты по убыванию значения в столбце age?**
  
Варианты ответов:
1) SELECT * ORDER BY age FROM table_name;
2) SELECT * FROM table_name ORDER BY age DESC;
3) ❌ SELECT * FROM table_name ORDER BY DESC age;
4) SELECT * FROM table_name WHERE age ORDER BY DESC;
  
### ИИ анализ
Вы выбрали неверный вариант. Правильный синтаксис для сортировки по убыванию — `ORDER BY column_name DESC`. Ключевое слово `DESC` должно стоять после имени столбца. В выбранном вами варианте порядок слов нарушен. Для сортировки по возрастанию используется `ASC` или оно опускается, так как это значение по умолчанию.  

# Вопрос 4. Тема: Основы SQL
  
**4. Как выбрать записи, где age меньше 30 или salary больше 50000, и отсортировать их по name?**
  
Варианты ответов:
1) ✅ SELECT * FROM table_name WHERE age < 30 OR salary > 50000 ORDER BY name;
2) SELECT * FROM table_name WHERE age < 30 OR salary > 50000 name;
3) SELECT * FROM table_name WHERE age < 30 AND salary > 50000 ORDER BY name;
4) SELECT * FROM table_name ORDER BY age < 30 OR salary > 50000, name;
  
### ИИ анализ
Вы выбрали верный ответ. В нём корректно используется логический оператор `OR` для выбора записей, удовлетворяющих хотя бы одному из условий, и предложение `ORDER BY` для сортировки результата. Важно отличать `OR` (логическое ИЛИ) от `AND` (логическое И), так как они возвращают разные наборы данных. Предложение `ORDER BY` всегда идёт в конце запроса.  

# Вопрос 5. Тема: Основы SQL
  
**5. Как выбрать записи, где salary больше 50000 и age больше 25?**
  
Варианты ответов:
1) SELECT * FROM table_name WHERE salary > 30000 NOT age > 25;
2) SELECT * FROM table_name WHERE salary > 30000 OR age > 25;
3) ✅ SELECT * FROM table_name WHERE salary > 30000 AND age > 25;
4) SELECT * FROM table_name WHERE salary > 30000, age > 25;
  
### ИИ анализ
Вы выбрали правильный ответ, используя логический оператор `AND`. Он возвращает только те строки, где выполняются оба условия одновременно. Оператор `OR` вернул бы записи, удовлетворяющие хотя бы одному условию, что не соответствует требованию задачи. В SQL для объединения условий в предложении `WHERE` используются ключевые слова, а не запятые.
