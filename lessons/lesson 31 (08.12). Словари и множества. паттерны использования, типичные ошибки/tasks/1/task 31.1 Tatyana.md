# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Музыкальная школа помечает, какой педагог репетировал с кабинетом последним. Заведующая распределяет расписание и проверяет свободные помещения. Создай словарь last_teacher, где ключ — кабинет, а значение — имя преподавателя. Затем выведи информацию для переменных today_room и free_room.

## Пример входных данных
```python
rooms = [
    ('piano-1', 'Ирина'),
    ('strings-2', 'Олег'),
    ('drums-1', 'Света'),
]
today_room = 'strings-2'
free_room = 'theory-3'
```

## Пример выходных данных
```text
strings-2 -> Олег
theory-3 -> свободен
```

## Критерии успешного выполнения
- Список пар преобразован в словарь last_teacher с ключом 'кабинет'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для today_room и free_room в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку rooms в цикле и наполняй словарь last_teacher.
- Используй множественное присваивание: key, value = запись из rooms.
- Для отсутствующих данных возвращай строку 'свободен'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
