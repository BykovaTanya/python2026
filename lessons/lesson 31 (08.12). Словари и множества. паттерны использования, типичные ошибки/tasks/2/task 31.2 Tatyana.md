# Задание 31.2. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Мутация словарей и обработка ошибок — Разбираем, как накапливать значения, безопасно удалять записи и не ловить KeyError при отсутствии данных.

## Условие задачи
Онлайн-школа считает количество попыток прохождения мини-курса учениками. Просуммируй попытки и удали ученика, который попросил перезачёт. Используй словарь attempts и аккуратные методы мутации.

## Пример входных данных
```python
logs = [('kate', 2), ('mike', 1), ('kate', 1), ('sofia', 3)]
finished = ['mike']
```

## Пример выходных данных
```text
... зависит от реализации
```

## Критерии успешного выполнения
- Повторяющиеся ключи накапливаются в словаре attempts с помощью get или setdefault.
- Удаление записей выполняется через pop с запасным значением 'нет попыток'.
- После удаления выводится обновлённый словарь.
- Код работает даже если в списке finished нет части ключей.

## Подсказки
- Создай пустой словарь attempts и заполняй его в цикле по logs.
- Для удаления используй pop(key, запасное значение), чтобы избежать KeyError.
- Перед удалением проверь наличие ключа через get.

## Частые ошибки и ограничения
- Не храни данные в списке, если уже построен словарь — это усложнит доступ.
- Не удаляй элементы через del без проверки.
- Не забывай выводить итог даже если удаления не было.
