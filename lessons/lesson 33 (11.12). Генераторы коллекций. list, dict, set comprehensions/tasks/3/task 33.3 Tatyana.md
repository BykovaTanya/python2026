# Задание 33.3. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Dict comprehensions» помогает за один проход собрать словарь с нужными значениями и фильтрами, сохраняя читаемость.

## Условие задачи
**Сложность**: <span style="color: #f9a825">● средний</span>. примерное время: 15–20 минут

Дан список `shipments`, где каждая запись — словарь с ключами `"city"`, `"items"`, `"delivered"`. Поле `"items"` хранит количество коробок, а `"delivered"` — сколько уже доставлено. Построй словарь `delivery_status`, в котором ключ — город, а значение — строка `"готово"`, если `delivered` равно `items`, иначе `"в пути"`. Добавляй в результат только города, у которых общее количество коробок не меньше 20.

## Пример входных данных
```
shipments = [
    {"city": "Москва", "items": 40, "delivered": 40},
    {"city": "Ярославль", "items": 18, "delivered": 10},
    {"city": "Пермь", "items": 25, "delivered": 12}
]
```

## Пример выходных данных
```
{'Москва': 'готово', 'Пермь': 'в пути'}
```

## Критерии успешного выполнения
- `delivery_status` создаётся одним dict comprehension.
- В словарь попадают только города с `items >= 20`.
- Формат значения строго `"готово"` или `"в пути"`.
- Исходный список `shipments` не изменяется.

## Подсказки
- Обращайся к полям как `shipment["items"]` и `shipment["delivered"]`.
- Используй тернарный оператор, чтобы выбрать нужную строку.
- Фильтр по количеству коробок ставь в конце comprehension.

## Частые ошибки и ограничения
- Не добавляй внешние циклы или вызовы `dict()` по результату.
- Следи, чтобы ключи словаря не повторялись.
- Не пиши больше пяти пунктов в одном списке.
