# Практическое задание 35.2

В предыдущем блоке вы научились объявлять функции, передавать аргументы и возвращать результат. Сейчас мы переходим к управлению аргументами: вы будете практиковаться в использовании значений по умолчанию для параметров, включая безопасную работу с изменяемыми типами данных. Также вы закрепите правила порядка передачи позиционных и именованных аргументов. Этот блок посвящен предотвращению распространенных ошибок (ловушка изменяемого значения по умолчанию) и развитию навыка чтения и написания вызовов функций с разными способами передачи данных.

## Задания

### Описание задачи
Вы помогаете разработать модуль для управления профилями пользователей в небольшом приложении. Ваша задача — написать функцию `update_profile`, которая обновляет информацию о пользователе.

Функция должна принимать следующие параметры:
1.  `user_id` (обязательный) — уникальный идентификатор пользователя (целое число).
2.  `name` (обязательный) — новое имя пользователя (строка).
3.  `settings` (необязательный) — словарь с настройками пользователя. Если параметр не передан, по умолчанию должен использоваться безопасный пустой словарь.
4.  `is_active` (необязательный) — флаг активности пользователя. По умолчанию должен быть `True`.
5.  `tags` (необязательный) — список тегов, связанных с пользователем. Если параметр не передан, по умолчанию должен использоваться безопасный пустой список.

Функция должна возвращать словарь с обновленной информацией о пользователе, включая все переданные данные.

**Важные детали:**
*   Вы должны избежать классической ошибки с изменяемыми значениями по умолчанию (список `tags` и словарь `settings`).
*   Функция должна корректно обрабатывать вызовы как с позиционными, так и с именованными аргументами.
*   Порядок параметров в объявлении функции должен позволять гибкие вызовы.

### Пример входных и выходных данных

**Пример 1: Простой вызов с позиционными аргументами**
```python
result = update_profile(101, "Анна")
print(result)
```
**Ожидаемый вывод:**
```python
{'user_id': 101, 'name': 'Анна', 'settings': {}, 'is_active': True, 'tags': []}
```
*Пояснение:* Переданы только обязательные аргументы. Для `settings` и `tags` использованы безопасные значения по умолчанию, для `is_active` — значение `True`.

**Пример 2: Вызов с именованными аргументами и изменением порядка**
```python
result = update_profile(name="Борис", user_id=202, is_active=False)
print(result)
```
**Ожидаемый вывод:**
```python
{'user_id': 202, 'name': 'Борис', 'settings': {}, 'is_active': False, 'tags': []}
```
*Пояснение:* Аргументы переданы по имени в произвольном порядке. `is_active` установлен в `False`.

**Пример 3: Вызов с передачей изменяемых объектов**
```python
my_settings = {"theme": "dark", "lang": "ru"}
my_tags = ["новичок", "премиум"]
result = update_profile(303, "Виктор", settings=my_settings, tags=my_tags)
print(result)
# Проверка, что исходные списки/словари не изменились
my_settings["theme"] = "light"
my_tags.append("тест")
print(result)
```
**Ожидаемый вывод:**
```python
{'user_id': 303, 'name': 'Виктор', 'settings': {'theme': 'dark', 'lang': 'ru'}, 'is_active': True, 'tags': ['новичок', 'премиум']}
{'user_id': 303, 'name': 'Виктор', 'settings': {'theme': 'dark', 'lang': 'ru'}, 'is_active': True, 'tags': ['новичок', 'премиум']}
```
*Пояснение:* Функция корректно принимает внешние изменяемые объекты. Изменение исходных `my_settings` и `my_tags` после вызова функции не должно влиять на данные внутри `result`. Это косвенно подтверждает, что функция работает с копиями или создает новые объекты, а не использует один общий объект по умолчанию.

### Критерии проверки и ограничения
1.  Функция должна называться `update_profile`.
2.  Параметры должны идти в порядке: `user_id`, `name`, `settings`, `is_active`, `tags`.
3.  Для изменяемых параметров `settings` и `tags` необходимо использовать `None`-паттерн для задания безопасных значений по умолчанию.
4.  Внутри функции, если `settings` равен `None`, ему должен присваиваться новый пустой словарь `{}`. Аналогично для `tags` — новый пустой список `[]`.
5.  Функция должна возвращать словарь с ключами: `user_id`, `name`, `settings`, `is_active`, `tags`.
6.  Запрещено использовать `*args`, `**kwargs`, распаковку, глобальные переменные и другие темы из будущих блоков.

### Решение задачи (псевдокод)
1.  Объяви функцию `update_profile` с параметрами: `user_id`, `name`, `settings`, `is_active`, `tags`.
2.  Для параметра `settings` установи значение по умолчанию `None`.
3.  Для параметра `tags` установи значение по умолчанию `None`.
4.  Для параметра `is_active` установи значение по умолчанию `True`.
5.  Внутри функции:
    a.  Если `settings` равен `None`, создай новый пустой словарь и присвой его переменной `settings`.
    b.  Если `tags` равен `None`, создай новый пустой список и присвой его переменной `tags`.
6.  Создай и верни словарь, где ключами будут строки `'user_id'`, `'name'`, `'settings'`, `'is_active'`, `'tags'`, а значениями — соответствующие переданные или вычисленные аргументы.

### Рекомендации

-   PEP8:
    -   Следите за отступами в 4 пробела для тела функции.
    -   После запятой, разделяющей параметры, ставьте один пробел (например, `def update_profile(user_id, name, settings=None, is_active=True, tags=None):`).
