# Практическое задание 35.1

Это задание относится к первому блоку темы "Функции". Вы уже знакомы с коллекциями и алгоритмами. Теперь вы переходите к основам функций: их объявлению, передаче аргументов и возврату значений. Это фундаментальный навык, который позволит вам структурировать код и избегать повторений. Данное упражнение закрепит понимание разницы между параметрами, аргументами и оператором `return`, подготавливая вас к изучению более сложных аспектов функций.

## Задания

### Описание задачи
Таня, вы помогаете разрабатывать простую систему для расчёта стоимости заказа в небольшом интернет-магазине. Вам нужно создать функцию, которая будет вычислять итоговую сумму заказа на основе позиций в корзине.

Напишите функцию `calculate_order`. Она должна принимать три **позиционных аргумента**:
1. `price_per_item` — цена одного товара (число).
2. `quantity` — количество товаров (целое число).
3. `discount` — скидка в процентах (число от 0 до 100). Скидка применяется к общей стоимости *до* её применения.

Функция должна:
1. Рассчитать общую стоимость заказа как `price_per_item * quantity`.
2. Применить скидку: вычесть из общей стоимости процент, заданный `discount`.
3. **Вернуть** итоговую сумму после скидки (число с плавающей точкой).
4. Не печатать результат внутри функции.

Ваша задача — корректно объявить функцию, использовать её параметры для вычислений и вернуть результат с помощью `return`.

### Пример входных и выходных данных

**Пример 1:**
*   **Входные данные для вызова функции:** `calculate_order(100, 3, 10)`
    *   `price_per_item = 100`
    *   `quantity = 3`
    *   `discount = 10`
*   **Что происходит:**
    1.  Общая стоимость: `100 * 3 = 300`
    2.  Сумма скидки: `300 * (10 / 100) = 30`
    3.  Итоговая сумма: `300 - 30 = 270`
*   **Ожидаемый возвращаемый результат функции:** `270.0`
*   **Как это может выглядеть в коде:**
    ```python
    total = calculate_order(100, 3, 10)
    print(total)  # Должно напечатать: 270.0
    ```

**Пример 2:**
*   **Входные данные для вызова функции:** `calculate_order(50, 5, 0)`
    *   `price_per_item = 50`
    *   `quantity = 5`
    *   `discount = 0` (скидки нет)
*   **Ожидаемый возвращаемый результат функции:** `250.0`

**Пример 3 (проверка граничного случая):**
*   **Входные данные для вызова функции:** `calculate_order(1, 10, 100)`
    *   `price_per_item = 1`
    *   `quantity = 10`
    *   `discount = 100` (полная скидка)
*   **Ожидаемый возвращаемый результат функции:** `0.0`

### Критерии проверки и ограничения
1.  Функция должна быть объявлена с использованием `def` и иметь имя `calculate_order`.
2.  Функция должна принимать ровно три **позиционных аргумента** в указанном порядке.
3.  Внутри функции **не должно быть** оператора `print` для вывода итоговой суммы.
4.  Функция должна использовать оператор `return` для возврата вычисленного числа.
5.  Для расчёта скидки используйте формулу: `итог = (price_per_item * quantity) * (1 - discount / 100)`.
6.  Не используйте темы из будущих блоков: именованные аргументы, значения по умолчанию, `*args`, `**kwargs`.

### Решение задачи (псевдокод)
1.  Объяви функцию с именем `calculate_order` и тремя параметрами: `price_per_item`, `quantity`, `discount`.
2.  Внутри функции:
    a.  Создай переменную `total_before_discount` и присвой ей результат умножения `price_per_item` на `quantity`.
    b.  Создай переменную `final_price`. Рассчитай её значение: `total_before_discount` умножить на `(1 - discount / 100)`.
    c.  Верни значение переменной `final_price` как результат работы функции.
3.  Вне функции (для тестирования):
    a.  Вызови функцию `calculate_order`, передав ей три числа в качестве аргументов.
    b.  Результат вызова функции можно сохранить в переменную или сразу передать в `print()`.

### Рекомендации
-   **Вынести в отдельную функцию:** В данном случае вся логика задачи и является функцией `calculate_order`. Дополнительные функции не требуются.
-   **PEP8:**
    -   Используйте осмысленные имена параметров, которые отражают их назначение (`price_per_item`, а не `p`).
    -   После объявления функции `def` оставляйте две пустые строки перед следующим блоком кода (например, перед вызовом функции для теста).
