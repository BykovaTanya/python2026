# Вопрос 1. Тема: Сложные запросы
  
**1. Какой оператор используется для объединения двух таблиц в SQL?**
  
Варианты ответов:
1) ✅ JOIN
2) UNION
3) COMBINE
4) MERGE
  
### ИИ анализ
Правильный ответ — JOIN. Оператор JOIN является основным в SQL для объединения строк из двух или более таблиц на основе связанного столбца между ними. Оператор UNION объединяет результаты запросов, а COMBINE и MERGE не являются стандартными SQL-операторами для этой цели. Важно различать объединение таблиц (JOIN) и объединение наборов результатов (UNION).

# Вопрос 2. Тема: Сложные запросы
  
**2. Как объединить таблицы Customers и Orders по полю CustomerID и выбрать все строки из таблицы Customers, даже если нет соответствующих записей в таблице Orders?**
  
Варианты ответов:
1) SELECT * FROM Customers LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID
2) ❌ SELECT * FROM Customers FULL JOIN Orders ON Customers.CustomerID = Orders.CustomerID
3) SELECT * FROM Customers RIGHT JOIN Orders ON Customers.CustomerID = Orders.CustomerID
4) SELECT * FROM Customers INNER JOIN Orders ON Customers.CustomerID = Orders.CustomerID
  
### ИИ анализ
Выбранный вариант (FULL JOIN) неверен. Для задачи выбрать все строки из таблицы Customers, даже без соответствий в Orders, необходим именно LEFT JOIN. LEFT JOIN возвращает все записи из левой таблицы (Customers) и совпадающие записи из правой (Orders), а при отсутствии совпадений поля из правой таблицы заполняются NULL. FULL JOIN возвращает все записи из обеих таблиц, что не соответствует условию.

# Вопрос 3. Тема: Сложные запросы
  
**3. Что вернёт запрос `SELECT Customers.Name, Orders.OrderID FROM Customers LEFT JOIN Orders ON Customers.CustomerID = Orders.CustomerID;`, если в таблице Customers есть клиенты без заказов?**
  
Варианты ответов:
1) ✅ Запрос вернёт всех клиентов (включая тех, у кого нет заказов) с пустыми значениями для OrderID
2) Запрос не вернёт результатов
3) Запрос вернёт только клиентов с заказами
4) Запрос вернёт только заказы, без клиентов
  
### ИИ анализ
Правильный ответ выбран верно. LEFT JOIN гарантирует, что все строки из левой таблицы (Customers) будут включены в результат. Для клиентов, у которых нет соответствующих записей в таблице Orders, значения столбцов из Orders (в данном случае OrderID) будут представлены как NULL. Это фундаментальное поведение левого внешнего соединения (LEFT OUTER JOIN).

# Вопрос 4. Тема: Сложные запросы
  
**4. Какой результат вернёт следующий SQL-запрос?**

```
SELECT Name FROM Customers
UNION
SELECT Name FROM Suppliers;
```
  
Варианты ответов:
1) Вернёт только совпадающие значения из обеих таблиц
2) ✅ Вернёт только уникальные имена из таблиц Customers и Suppliers
3) Объединит результаты, но добавит NULL вместо отсутствующих значений
4) Объединит результаты обеих таблиц, включая дубликаты
  
### ИИ анализ
Правильный ответ выбран верно. Оператор UNION по умолчанию объединяет результаты двух или более SELECT-запросов, автоматически удаляя дублирующиеся строки из финального набора. Таким образом, результат будет содержать список всех уникальных значений из столбца Name обеих таблиц. Для включения дубликатов необходимо использовать UNION ALL.

# Вопрос 5. Тема: Сложные запросы
  
**5. Какой результат вернёт следующий SQL-запрос?**

```
SELECT ProductName FROM Products
EXCEPT
SELECT ProductName FROM DiscontinuedProducts;
```
  
Варианты ответов:
1) Вернёт список всех товаров, включая устаревшие
2) Вернёт только устаревшие товары
3) ✅ Вернёт товары, которые есть в Products, но отсутствуют в DiscontinuedProducts
4) Вернёт только товары, которые присутствуют в обеих таблицах
  
### ИИ анализ
Правильный ответ выбран верно. Оператор EXCEPT (в некоторых СУБД называется MINUS) возвращает строки из первого набора данных, которые отсутствуют во втором. В данном контексте запрос вернёт названия продуктов, которые есть в текущем ассортименте (Products), но не числятся в списке снятых с производства (DiscontinuedProducts). Это полезно для получения "активных" товаров.
